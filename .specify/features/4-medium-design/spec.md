# 기능 명세서: Medium 스타일 디자인 완성

**작성일**: 2025-11-15
**상태**: 초안
**우선순위**: 높음 (출시 주간)
**관련 헌장 원칙**: 원칙 2 (디자인 일관성 및 Medium 스타일 미학)

## 개요

### 목적
블로그 전체를 Medium의 깔끔하고 미니멀한 디자인 언어로 통일하여 읽기에 집중할 수 있는 사용자 경험을 제공합니다.

### 배경
홈페이지와 PostCard는 이미 Medium 스타일로 업데이트되었으나, Header, Footer, 카테고리 페이지, 블로그 상세 페이지가 여전히 구식 그라디언트/컬러풀 디자인을 사용 중입니다.

### 범위

**포함사항:**
- Header 컴포넌트 Medium 스타일 업데이트
- Footer 컴포넌트 Medium 스타일 업데이트
- 카테고리 페이지 리스트 레이아웃 변경
- 블로그 상세 페이지 스타일 개선
- 구독 페이지 심플화

**제외사항:**
- 기능 변경 (순수 스타일 작업만)
- 새로운 컴포넌트 추가

## 현재 상태 vs 목표

### Header (현재 문제점)
- 그라디언트 버튼 사용 (`bg-gradient-to-r from-blue-600 to-purple-600`)
- 복잡한 배경 효과 (`backdrop-blur`)
- 너무 많은 색상

**목표**:
- 단색 검정/흰색 배경
- 심플한 pill-shaped 버튼
- 최소한의 테두리만 사용

### Footer (현재 문제점)
- 어두운 배경 (`bg-gray-900`)
- Medium과 상반된 느낌

**목표**:
- 밝은 배경 (`bg-white`) + 얇은 상단 테두리
- 깔끔한 링크 목록
- 불필요한 섹션 제거

### 카테고리 페이지 (현재 문제점)
- Grid 레이아웃 (카드 스타일)
- 회색 배경 (`bg-gray-50`)

**목표**:
- 리스트 레이아웃 (세로 나열)
- 흰색 배경
- 테두리로 구분

### 블로그 상세 페이지 (현재 문제점)
- 그라디언트 배경 (커버 이미지 없을 때)
- 컬러풀한 Premium 배지

**목표**:
- 깔끔한 커버 이미지 또는 배경 없음
- 심플한 ⭐ 아이콘으로 대체

## 디자인 원칙 (헌장 기반)

1. **타이포그래피**: 제목은 세리프 폰트, 본문은 산세리프
2. **색상**: 검정(#000), 회색(#757575), 흰색(#FFF) 위주
3. **레이아웃**: 여백 충분, 콘텐츠 중심
4. **구분선**: 그림자/그라디언트 대신 `border-b border-gray-200`
5. **버튼**: 둥근 pill 형태 또는 간단한 텍스트 링크

## 기능 요구사항

### FR1: Header 업데이트
- **로고**: 검정 텍스트 ("CryptoTitan")
- **네비게이션**: 심플한 텍스트 링크 (호버 시 밑줄)
- **로그인 버튼**: 테두리 버튼 (`border border-gray-900`)
- **구독 버튼**: 검정 배경 pill 버튼 (`bg-gray-900 text-white rounded-full`)
- **배경**: 흰색 + 하단 얇은 테두리

### FR2: Footer 업데이트
- **배경**: 흰색, 상단 테두리만
- **레이아웃**: 단순 2-3 컬럼
- **뉴스레터 폼**: 인라인 스타일 (최소화)
- **링크**: 회색 텍스트, 호버 시 검정

### FR3: 카테고리 페이지 리스트 레이아웃
- **레이아웃**: Grid 제거 → 1열 리스트
- **PostCard**: 기본 variant (가로 레이아웃)
- **배경**: 흰색
- **구분**: 각 카드 간 `border-b`

### FR4: 블로그 상세 페이지
- **커버 이미지**: 있으면 전체 폭 표시, 없으면 **흰색 배경만** (그라디언트 완전 제거)
  - 커버 없을 때: 제목부터 바로 시작 (Medium 스타일)
  - 최소 여백만 유지 (`pt-12`)
- **제목**: `font-serif text-4xl` 큰 세리프 폰트
- **메타 정보**: 작은 회색 텍스트
- **본문**: `prose prose-lg` 사용, 최대 너비 제한
- **Premium 배지**: 단순 ⭐ 텍스트

### FR5: 구독 페이지 심플화
- **가격 카드**: 그라디언트 제거, 테두리만 사용
- **버튼**: 검정 배경 pill 버튼
- **배경**: 흰색

## 성공 기준

### 정량적 지표
1. 모든 페이지에서 그라디언트 0개
2. 사용 색상 5가지 이하 (검정, 흰색, 3단계 회색)
3. 버튼 스타일 2가지 이하 (테두리 + 단색)

### 정성적 지표
1. 사용자가 "깔끔하다", "읽기 좋다" 피드백
2. Medium과 유사한 느낌
3. 브랜드 일관성 유지

## 테스트 시나리오

### 수용 테스트 1: Header 확인
1. 홈페이지 상단 Header 확인
2. **기대 결과**: 그라디언트 버튼 없음, 깔끔한 흰색 배경

### 수용 테스트 2: 카테고리 페이지
1. 카테고리 페이지 접속
2. **기대 결과**: 리스트 레이아웃, 카드 아님

### 수용 테스트 3: 블로그 상세
1. 블로그 글 페이지 접속
2. **기대 결과**: 커버 이미지 없으면 그라디언트 배경 없음

## 엣지 케이스

### EC1: 매우 긴 제목
- **상황**: 블로그 제목이 3줄 이상으로 넘어가는 경우
- **처리**: 제목은 자동 줄바꿈, 최대 라인 제한 없음 (Medium 스타일)
- **폰트 크기**: 모바일에서는 `text-3xl`로 축소

### EC2: 반응형 네비게이션
- **상황**: 태블릿 화면 (768px-1024px)
- **처리**: 데스크톱과 동일한 가로 네비게이션 유지
- **버튼**: 패딩 살짝 축소 (`px-4 py-2`)

### EC3: 다크 모드 미지원
- **상황**: 사용자 시스템이 다크 모드 설정
- **처리**: 현재 버전은 라이트 모드만 지원
- **향후**: Phase 3에서 다크 모드 추가 고려
